# Task 5 - Capture and Analyze Network Traffic Using Wireshark
Wireshark network traffic capture and protocol analysis (DNS, TCP, HTTP).

## Objective
To capture live network packets and analyze different network protocols using Wireshark.

## Tool Used
- Wireshark

## Steps Performed
1. Installed Wireshark on Windows.
2. Selected the active Wi-Fi network interface.
3. Started packet capture.
4. Generated network traffic by browsing websites.
5. Stopped the capture after one minute.
6. Applied filters to identify different protocols such as DNS, TCP, and HTTP.
7. Saved the packet capture as a .pcapng file.

## Protocols Identified

### DNS
- Used to translate domain names into IP addresses.

### TCP
- Provides reliable communication between devices.

### HTTP
- Used for transferring web pages and other web content.

### UDP
- A connectionless protocol used for fast communication where speed is more important than guaranteed delivery.

## Outcome
Successfully captured live network traffic and analyzed different protocols using Wireshark. This task improved my understanding of packet capture, protocol analysis, and basic network troubleshooting.

## Conclusion
Wireshark is a powerful network protocol analyzer that helps monitor, inspect, and troubleshoot network communication by capturing live packets.
